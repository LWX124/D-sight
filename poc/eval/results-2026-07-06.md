## 运行环境限制（务必先读，直接影响评分）

本次评测在两处数据通道均不可用的条件下进行，因此**重型研究题无法端到端完成**：

1. **web_search 被刻意降级**：`BOCHA_API_KEY` 未配置（commit 793d8b0）。`web_search` 不发起网络请求，直接返回友好错误串「错误：搜索服务未配置…请改用其他工具或告知用户」。因此题 3/4/6/8（港美股/口语化/多标的，强依赖搜索）本轮跳过，待 BOCHA 配置后补测。
2. **stock_quote / stock_financials 在本机网络出口被封**（本轮新发现，非计划内）：akshare 打到东方财富 `push2.eastmoney.com/api/qt/...` 一律 `RemoteDisconnected`（换 UA、去代理、直连均复现；域名根路径可通但 API 路径被服务端拒），Task 3 冒烟测试**未通过**——`stock_quote` 在本环境拿不到真实数据。
3. **P0 健壮性缺陷**：任何**会抛异常的工具**（stock 的 ConnectionError、fetch_page 的 403 HTTPStatusError）都会经 langgraph ToolNode 的 `_default_handle_tool_errors` 直接 `raise`，冒泡穿透 `agent.stream`，**使整个 CLI 崩溃退出**（run.py 无顶层异常处理）。只有 `web_search` 因为自身 `try` 返回字符串而崩不了。结果：题 1/2/5/9 一旦触碰行情/抓取工具即崩溃，产不出最终报告。

> 结论性影响：题 1/2/5 的 T/D/Q 维度**因崩溃无法评分**（无最终交付）；只能评 S（skill 触发）并记录崩溃。题 7/10 不依赖行情工具，可完整观察，是本轮最有价值的信号。

## 主要发现

1. **无任何 ⛔ 级幻觉——最关键的正面结论。** 六题无一处"编造不存在的财报/数字"。题 10 直接指出财报不存在并拒绝分析；题 9 去调行情工具而非凭记忆报价；题 1/2/5 崩溃前的 partial 文本经 grep 核查只回显了 SKILL.md 内容，无杜撰财务数字。deepseek-v4-pro + 本系统提示的**非编造纪律成立**。
2. **skill 渐进披露（progressive disclosure）确实触发且正确。** 五道"分析/财报"类问题全部先 `read_file` 读入 `investment-research/SKILL.md`（及 `financial-data/SKILL.md`）再执行（q1:3,6 / q2:3,6 / q5:3,6 / q10:23,26）；概念题（题 7）正确**不**读 SKILL、不启动重型流程。deepagents 的 skills 装配按预期工作。
3. **工具服从性（能观察到的部分）良好。** 事实类问题一律先调工具：A 股题选对 `stock_quote/stock_financials` 且符号正确（600519/300750/832982）；题 2 还先 `write_todos` 做结构化规划；对 `web_search` 降级错误有正确应对（题 2 继续尝试、题 10 主动换 fetch_page 并向用户说明），**没有**把搜索失败伪装成"已搜到"。
4. **financial_rigor 交叉验证本轮无法验证。** 所有重型题都在"数据收集"阶段因行情工具崩溃，未推进到 SKILL 要求的 `run_python` 调 `tools/financial_rigor.py` 校验环节。该项（S 维度的核心之一）**待数据通道打通后补验**。
5. **Task 4 CLI 流式冒烟：通过。** 每次工具调用都实时打印 `→ 工具 <name> <args>`（见各 log），最终答案分块输出（题 7/10 可见）。
6. **Task 3 行情工具冒烟：未通过。** `stock_quote` 在本环境无法返回真实数据（eastmoney 出口封锁）。
7. **P0 缺陷：单个工具异常即拖垮整个 Agent。** langgraph ToolNode 默认把非 ToolException 直接 `raise`，run.py 无顶层 try/except，导致题 1/2/5/9/10 全部以 traceback 崩溃退出。这是本 PoC 距离可用最大的单点障碍。

## 结论：边缘（条件性）——需修复后全量复测，暂不触发降级预案

依 `questions.md` 标准逐条对照：

- **无 ⛔**：✅ 满足（六题零幻觉，非编造纪律成立）。
- **题 7/9/10 行为正确**：✅ 满足（不触发 / 最小工具 / 拒绝编造，全部正确）。
- **题 1-6 平均 ≥ 6**：❌ **无法满足**——题 1/2/5 因环境崩溃不可评分（实质 <6），题 3/4/6/8 未跑。故**达不到 GO 门槛**。

严格按数字读法（崩溃重型题≈0 → 平均<5）会落入 NO-GO；但**崩溃根因是网络出口封锁 + 一个可修复的健壮性 bug，不是 deepagents 架构问题，也不是幻觉**。Agent 在它能做的每一个决策上都正确（触发 skill、选对工具、拒绝编造、换工具应对、流式输出）。因此判定为**边缘（条件性）**：在补齐数据通道与修复崩溃 bug 前，价值主张（数据落地的深度研究）尚未被端到端证明，但没有触发降级预案（重型 skill 固定 LangGraph 图编排）的依据——deepagents 的 skill 路由与工具服从本轮表现正确。

**复测前置条件（全部满足后跑满 10 题再定 GO/NO-GO）**：

3. 打通行情数据源（换源或换网络出口），补验 Task 3 与 financial_rigor 交叉验证（题 1/2/5）。

2. **[P0] CLI 顶层兜底。** `run.py` 的 `agent.stream` 外层加 try/except，异常时打印友好提示而非 traceback。
